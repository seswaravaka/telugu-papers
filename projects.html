Architecture & Implementation

Platform Design

Designed and developed a centralized management platform for XRAN products using PostgreSQL as the primary database, Spring Boot–based backend services, and REST APIs, Kafka events, and streaming connections for data ingestion and communication.

The platform is deployed as containerized microservices on Kubernetes-based clusters, including k3s and OpenShift.

Service Development

  • Designed service architecture and inter-service communication patterns
  • Packaged services as Docker images
  • Designed REST APIs and data exchange contracts
  • Used Kafka events for inter-service communication, achieving a near event-driven architecture

Security & Access Management

  • Designed and implemented OAuth2 / OpenID Connect authentication across all user-facing services
  • Used Istio service mesh for gateway functionality
  • Integrated authentication and authorization using oauth2-proxy
  • Integrated and customized Keycloak to meet product-specific requirements
  • Supported customer directory integrations (LDAP, Active Directory, external IdPs)
  • Implemented SAML integration for customer environments (e.g., Verizon)
  • Implemented mutual authentication using certificates

APIs & Integrations

  • Published APIs using OpenAPI 3.0 / Swagger
  • Developed and shared SDKs for customer integrations
  • Collaborated with system integrators including Cisco, SoftBank, and Wind Tre
  • Developed SOAP APIs and SDKs using JAX-WS (deprecated in later releases)

Operations, Monitoring & Automation

  • Implemented automated configuration management with backup and continuous validation
  • Integrated Prometheus for performance metrics collection
  • Used Grafana for visualization and dashboards
  • Implemented centralized logging, fault, and alarm handling
  • Integrated NGINX for front-end traffic handling

Configuration Management & Orchestration

  • Integrated with SaltStack using CherryPy REST APIs
  • Used Ansible jobs to invoke automation playbooks
  • Playbooks triggered Helm chart deployments on targeted clusters

UI & Real-Time Communication

  • Used Angular for UI development
  • Implemented WebSocket and Server-Sent Events (SSE) for real-time UI updates
  • Used Kafka listeners and Kafka Streams for event consumption
  • Implemented Kafka producers where required

Release Management & Delivery

  • Release-based development model
  • Each release treated as an individual project with defined features
  • Led design and delivery over a 10-year period
  • Delivered 30+ major and minor releases
