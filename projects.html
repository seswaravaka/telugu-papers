XRAN Management Platform

I designed and developed a centralized management platform for XRAN products, providing unified control and visibility across the entire product lifecycle.

The platform supports configuration management, performance monitoring, fault and alarm handling, log management, system monitoring, and software installation and upgrades.

Architecturally, the system is built using PostgreSQL and Spring Boot–based microservices, with REST APIs, Kafka events, and streaming connections enabling scalable data ingestion and communication.

All services are containerized and deployed on Kubernetes‑based clusters, including k3s and OpenShift.

Architecture & Implementation

I designed service boundaries, inter‑service communication patterns, and data exchange contracts. Services were packaged as Docker images and deployed as microservices.

Kafka was used extensively for inter‑service communication, enabling a near event‑driven architecture across the platform. REST APIs and streaming connections supported both synchronous and real‑time data flows.

APIs were published using OpenAPI 3.0 / Swagger, and SDKs were developed and shared with customers to support external integrations with the centralized management system.

Security & Access Management

Security was a core design principle. I implemented OAuth2 and OpenID Connect authentication across all user‑facing services. Istio was used as the service mesh gateway, with oauth2‑proxy providing authentication and authorization enforcement.

The platform integrated with Keycloak, customized for product‑ specific requirements, and supported customer directory integrations including LDAP, Active Directory, and external identity providers.

SAML integration was implemented for customer environments such as Verizon, along with mutual TLS authentication using certificates.

APIs & Integrations

I collaborated closely with customers and system integrators, including Cisco, SoftBank, and Wind Tre, to support integrations with external systems.

In addition to REST APIs, SOAP APIs and SDKs were developed using JAX‑WS. These interfaces were later deprecated in favor of modern REST‑based integrations.

Operations, Monitoring & Automation

I designed and implemented automated configuration management with backup and continuous validation, ensuring that correct configurations were consistently applied across all managed elements.

Prometheus was used to collect performance metrics from all nodes, with Grafana providing visualization and dashboards. Centralized logging, fault handling, and alarm management were built into the platform.

NGINX was integrated for front‑end traffic handling and request routing.

Configuration Management & Orchestration

The platform integrated with SaltStack using CherryPy REST APIs to support configuration and orchestration workflows.

Ansible jobs were used to invoke automation playbooks, which in turn triggered Helm chart deployments on targeted Kubernetes clusters.

UI & Real‑Time Communication

User interfaces were developed using Angular. Real‑time updates were delivered to the UI using WebSockets and Server‑Sent Events (SSE).

Kafka listeners, Kafka Streams, and producers were implemented to support real‑time event processing and data propagation.

Release Management & Delivery

XRAN product development followed a release‑based model, with each release treated as an individual project and owned end‑to‑ end by the team.

Over a ten‑year period, I managed the design and delivery of more than 30 major and minor releases, coordinating architecture, development, and delivery across teams.
